## 13. Operational Runbook

### 13.1 Common Failure Modes

| Symptom | Likely Cause | Resolution |
|---------|--------------|------------|
| Users report "KYC stuck in SUBMITTED for 24h+" | External verification service (identity/bank/demat) is down or slow | Check service health dashboards; if down, trigger manual DLT recovery workflow; notify users via email of delay |
| Kafka listener crashes repeatedly; DLT filling up | Message deserialization error or infinite loop in listener | Review latest DLT messages via: `kafka-console-consumer --topic DEV.KYC.{CATEGORY}.IN.DLT --from-beginning`; fix code; redeploy |
| S3 upload fails with "Access Denied" | AWS IAM credentials expired or wrong bucket | Verify IAM role has s3:PutObject permission; check bucket name in config; rotate credentials if needed |
| API returns 401 Unauthorized for valid tokens | JWT validation failed; token expired or signature mismatch | Check auth service is running; verify JWT secret in vault; check system clock skew (NTP) |
| Database connections maxed out; service slow | Connection pool exhausted | Increase `spring.datasource.hikari.maximum-pool-size` in config; check for leaked connections in code; restart service |
| Redis cache down; app proceeds but slow | Redis cluster unavailable | Failover to read replica; or restart; app continues without cache (graceful degradation) |
| Manual approval email not sent | Notification service integration broken | Check notification service is running; verify email config; resend via admin panel |
| Status derivation shows INCOMPLETE even though all items APPROVED | Status rules JSON is stale or invalid | Verify status-rules.json loaded correctly; check logs for parsing errors; restart service to reload config |
| DLT messages accumulate; not being retried | No DLT recovery process running | Ops must manually review & republish to IN topic; implement automated DLT reprocessor (future task) |
| Duplicate KYC items created for same category | Race condition in save logic | Check idempotency key validation; verify request deduplication in gateway; fix race in service if found |
